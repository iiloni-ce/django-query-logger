# Product Guidelines

## Prose Style
- **Professional & Technical**: Documentation should be clear, concise, and technically accurate.
- **Direct**: Use active voice and avoid fluff.

## Branding
- **Color Palette**: Dark mode by default for developer tools, using accessible colors for query highlighting.
- **Tone**: Pragmatic and tool-oriented.

## UX Principles
- **Realtime Feedback**: Ensure minimal latency between query execution and UI update.
- **Clutter Reduction**: Use sensible defaults for query filtering to avoid overwhelming the user.
- **Keyboard Friendly**: Support common developer shortcuts for navigation and search.
